• ASSESSMENTS

1a. External Network Pentest
* Assessing an organization's security from outside looking in.
* Methodology focuses heavily on Open-Source Intelligence(OSNIT) Gathering.
* Typically lasts 32-40 hours with another 8-16 for reports

1b. Internel Network Pentest
* Assessing an organization's security from inside of the network.
* Methodology focuses heavily on Active Directory Attacks.
* Typically lasts 32-40 hours with another 8-16 for reports

2. Web Application Pentest:
* Assessing an organization's web security application.
* Methodology focuses heavily on web based attacks and the OWASP testing guidelines.
* Typically lasts 32-40 hours with another 8-16 for reports

3. Wireless Pentest
* Assessing an organization's wireless network security.
* Methodology depends on wireless type being used(guest vs WPA2-PSK vs WPA2 Enterprise).
* Typically lasts 4-8 hours per SSID with another 2-4 for reports


4. Physical Pentest and Social Engineering
* Assessing an organization's physical security and/or end-user training.
* Methodology depends on tasks and goals.
* Typically lasts 16-40 hours with another 4-8 for reports

    • REPORT WRITING
* delivered within a week after engagement ends.
* should highlight both non-technical(executive) and technical findings.
* Recommendations and remediations should be clear to both executives and technical staff.


    • DEBRIEF
* A debrief walks through your report findings. This can be with technical and non-technical staff present.
* It gives oppurtunities for the client to ask questions and address any concerns before a final report is released.
